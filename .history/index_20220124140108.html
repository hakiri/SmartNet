About

Executive Summary

The project aims at designing and developing a smart network architecture for supporting future cognitive and autonomic edge network end-to-end service orchestration, control and management. Such an intelligent network communication system provides the necessary infrastructure that intertwine key enabling technologies to create a single, coherent and unifying network control framework.

  • The control framework should support future network design, by combining smart connectivity, data analytics (AI/ML), distributed communication, automated and greatly cost reduced network operation.
  • The proposed architecture offers programmable analytics to the application layer through open interfaces, instantiate service intelligence at the edge as well as across cores.
  • The proposed ideas will design and deliver a programmable and customizable Network Computing Points (NCPs) for autonomously managing and controlling Edge network.

The proposed ideas will design and deliver a programmable and customizable Network Computing Points (NCPs) for autonomously managing and controlling Edge network. The autonomous capabilities stem from a novel three-layered Software-Defined architecture comprising:

  • The meta-control layer, which used for NCPs programmability and customization
  • The management layer, which is used for resource management on the NCPs.
  • The application layer that enables one or more applications tasks to execute on the potentially lightweight virtualized resources (micro-service) of the NCP.

Novel outcomes expected from this research include the realization of a 3-layered Software-Defined architecture, application of product line engineering approaches in automating the protocol stack customization, new approaches for coordinated analytics on edge platforms, and resource allocation algorithms for local and global resource management that are informed by the coordinated analytics for both wireless and mobility patterns of NCPs.

Develop advanced AI/ML Planning and Placement Techniques to implement the concept of fully automated Fog Networks.

Objective 1 implements a fully automated service deployment and resource/capacity planning mechanisms for fast path forwarding across ultra-low latency SDN-enabled virtualized Fog infrastructure.

  • Offer predictive behaviours on the network, prevent the impact of failures, support highly available and fast adapt network configuration at the target performance levels requested by the applications.
  • Applying Big Data intelligent networking to offer mechanisms for real-time analysis and off-line analysis of massive historical data.
  • Supporting distributed monitoring architectures for network management, analytics, visualization for network traffic and intelligent routing and caching.

Implement Programmable & Customizable Network Computing Points (NCPs)

Objective 2 delivers a programmable and customizable Network Computing Points (NCPs) that combines Intelligence Defined Network, mathematical programming, and lightweight virtualization technologies (i.e. containers, micro-services, unikernel) to autonomously manage, control, and provision edge network resources.

  • Enforcing Fog in-network programmability using Programmable Data Plane (P4 approach) and Deeply Programmable Network (DPN) Virtualization towards full autonomic network processing, automation and orchestration.
  • Offering a fully virtualised network control for increased flexibility by using lightweight virtualisation technologies (containers, unikernel), fast path networking with XDP and eBPF, and minimalistic distributions of general-purpose OSes (OS tinyfication).
  • Objective 2 will be highly innovative because they provide us with new information on the SDN-enabled Segment Routing approach across Ultra-Reliable Low Latency Communication (URLLC) using advanced QoS techniques (IEEE TSN, IETF DetNet)

Design and Deliver of a SDN-enabled Infrastructure-as-Code

Objective 3 aims at delivering a SDN-enabled Infrastructure-as-a-Service to autonomously manage, control, and provision edge network resources. Such an infrastructure should enable programmable and customizable Network Computing Points (NCPs).

  • The NCPs should offer in-network computation for intelligent networking and dynamic resource management and monitoring, by combining Fog Computing meta-data and Multi-access Edge Computing (MEC).
  • NCPs will support fast path networking (i.e. eXpress Data Path(XDP) and extended Berkeley Packet Filter (eBPF)).
  • Supporting end-to-end Fog network virtualization and resource management for fast path forwarding..

.

Develop Blockchain Techniques to Enforce Federated Fog network Integrity and Resiliency

Combining Blockchain distributed ledgers and reinforcement learning techniques to address network resilience and fault tolerance, design federated SDN networks, enforce network integrity and resiliency that operate under uncontrolled conditions (e.g. congestion).

  • Introducing decentralized learning system so that federated learning makes the fog network fully decentralize, more flexible and efficient. AI/ML techniques will be used to design distributed trust systems (in form of virtualized network functions), enforce network security for anomaly detection and mitigation.
  • Enforce network security and trust using machine learning based (ML) Blockchain algorithms to identify security concerns (intrusion, vulnerabilities, etc.), eliminate risk mitigation, intercept distributed attack.
  • Keeping autonomic network management by providing security rules and policies (i.e. securely recovering from runtime vulnerabilities or attacks and restore the network functions to an operational state).

Services

Work Packages

Work Package 1: Project coordination

The WP1 is responsible for all the project administrative tasks, technical coordination activities, and innovation management. Additionally, WP1 includes series of complementary tasks to support Data Management Plan (DMP), which focus on innovative claims and deliverables and deal with multiple Intellectual Property Rights (IPR) management (to access software components and algorithms, experimental data access and sharing policies, and material for education and outreach purposes).

Work Package 2: Requirements, Scenarios and Architecture

The WP2 identifies the of the project requirements, scenarios and use cases which will guide the project technical activities. A set of initial business and communication requirements will be identified by the project team, and then will be used to develop initial scenarios that will guide all technical decisions and system architectural design in order to specify the functional architecture definition and top-level design.

Work Package 3: Advanced AI/ML Provisioning Techniques

The WP3 provide the advanced AI/ML, techniques and algorithms for supporting reinforcement learning for future communication networks (initial design, data filtering, features extraction, structure modelling and training). In addition to the reinforcement learning approach, WP 3 applies Big Data and Data mining intelligent networking techniques for enabling real-time analysis, off-line analysis of massive historical data, distributed monitoring and traffic visualization, advanced SLA/QoS management, and intelligent routing and caching. Useful insights will be generated based on the automatic analysis of all that data.

Work Package 4: Virtualized Network and Resource Management

The focus of WP 4 use useful insights generated based on the automatic analysis of all data provided by the advanced AI/ML approach to offer a fully autonomic, programmable and virtualized network control, management and orchestration for increased flexibility and agility. End-to-end ultra-reliable low latency communication is enhanced through SDN-enabled Segment Routing along with emerging deterministic networking approaches (IEEE TSN and IETF DetNet). Additionally, this WP should build on lightweight virtualisation and containerization technologies (i.e. dockerization, unikernel) and fast path networking (i.e. XDP and eBPF) as well.

The lightweight virtualisation should be used to build a programmable and customizable Network Computing Points (NCPs) based on multi-layered Software-Defined Infrastructure-as-a-Service for autonomously managing and controlling and unifying Fog Computing and Multi-Access Edge Computing (MEC). Besides, real-time networks such as TSN (Time Sensitive Networks) and accelerator aware provisioning (XDP) should be used to enabled decentralisation, efficiency, interoperability, security and extensibility.

Work Package 5: Blockchain-based Network Security and Trust

The WP 5 aims at designing end-to-end distributed control plane using the principal of Blockchain distributed ledgers across separately managed SDN networks. This WP uses AI/ML outputs from WP2 for producing decentralized intelligences (either on-chain for basic information or off-chain in case of extra attachments) by introducing decentralized learning system so that federated learning could make the system fully decentralize, more flexible and efficient. Additionally, AI/ML techniques in WP 2 will be used to design distributed trust systems (in form of virtualized network functions), enforce network security for anomaly detection and mitigation.

Work Package 6: Validation and Integration

The WP 6 relates to the integration effort in combining the technologies developed by various partners to carry out tests and validation. WP 6 focus also on the integration of the Edge network components across cores and with other complementary technologies to realize the goals of the project, with particular focus on autonomic end-to-end service orchestration and self-adaption as well as data visualization. This task is illustrated through the development number of demonstrators from real-life use cases to highlight the value of the project innovation.

Work Package 7: Dissemination, standardization, exploitation and technoeconomic monetarization

This WP7 focus on ensuring the proper data dissemination and exploitation of the project results. This WP is very important as it promotes the use of data models, in-situ analytics, data models, structure modelling and training to raise public and scientific awareness of the EU-funded project. Furthermore, this WP assures the standardization of the main outcomes of the project.
